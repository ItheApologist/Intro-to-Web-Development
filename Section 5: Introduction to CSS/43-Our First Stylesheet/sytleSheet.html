One issue with the above example is that you have to place the styling for each tag, making work cumbersome. The other issue with this is that all the code, both the HTML structure and the styling, will be contained within the same file. This again makes work cumbersome opposed to using two files to separate both structure and styling.

The second method of styling is to use style tags. You will notice on our page that both h1 headers are purple. However, if you look at the code, you won't see any styling in the h1 tags. That's because we used the Style Tag method to create the styling. To do this, we insert the style tag into the HTML header. We call the type as text/css, then write the CSS syntax to call all h1 headers to be purple. Check the source page for the example. I've also put in an li styling using this method resulting in all unstyled li tags (meaning they don't have any specified styling in them) to turn orange.

One thing we have been doing wrong thus far has been to write our CSS in the same file as our HTML. By doing so, we are not following the methodology of keeping our structure and styling separate. To resolve this, we can use a link tag in the HTML header that references our local HTML file.
