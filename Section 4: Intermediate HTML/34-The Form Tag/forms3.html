The only thing the second form does is refresh the page. Remember, we have two items that need to be specified for a form to work: The action and the method. The action is where the form data is sent to and the method is a get/post. By default, if no action is specified, it will be sent to its own location. If a method isn't specified, it will use get to pull the data. Hence why the page refreshes. It first sends the data to the same location. This HTML page has nothing to store that data, so nothing happens. Then a get is called that pulls the page again, causing a refresh.
